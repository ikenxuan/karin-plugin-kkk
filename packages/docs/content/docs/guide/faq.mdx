<Accordion title="解析失败或返回空？">

1. 检查 Cookies 是否有效（可尝试重新扫码登录）
2. 检查网络是否正常
3. 部分内容可能需要登录态才能获取

</Accordion>

<Accordion title="B站解析遇到 -352 错误？">

这是 B站的风控机制，需要手动完成极验验证。

<Callout type="info">
使用 <LinkPreview url="/geetest" className="font-bold text-fd-primary hover:underline">极验验证器</LinkPreview> 工具手动完成验证，验证成功后结果会自动复制。
</Callout>

**使用步骤：**
1. 打开 <LinkPreview url="/geetest" className="font-bold text-fd-primary hover:underline">极验验证器</LinkPreview>
2. 输入错误信息中的 `gt` 和 `challenge` 值
3. 点击「生成验证码」完成验证
4. 验证结果会自动复制，重新触发解析即可

</Accordion>

<Accordion title="视频太大发不出去？">

- 使用自动画质模式：`videoQuality: adapt`（抖音/小红书）或 `videoQuality: 0`（B站）
- 调低 `maxAutoVideoSize` 值
- 或直接设置较低画质如 `720p`

</Accordion>

<Accordion title="推送不生效？">

1. 确认 `push.switch` 已开启
2. 检查 `pushlist.yaml` 中 `group_id` 格式：`群号:机器人账号`
3. 确保机器人在目标群中
4. 查看日志是否有报错

</Accordion>

<Accordion title="为什么开了烧录弹幕后视频画质变差了？">

开启弹幕烧录后，视频需要进行**重新编码 (Re-encode)** 以将字幕“印”在画面上。

- **有损过程**：重新编码本质上是一个有损过程，必然会导致一定程度的画质损失。
- **码率控制**：插件默认会尝试匹配原视频的码率，但为了平衡处理速度和文件大小，可能会有一定的压缩。
- **改善方法**：
  - 尝试在配置中将 `videoCodec` 设置为 `h265` 或 `av1`（如果硬件支持）。
  - 确保没有开启过低的比特率限制。

</Accordion>
