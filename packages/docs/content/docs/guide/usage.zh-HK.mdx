## 解析功能

插件而家支持兩類入口：自動命中分享連結，同引用消息手動解析。

### 自動解析

預設情況下，插件會監聽消息內容。只要平台開關已打開，命中支持嘅連結、短鏈、BV 號、AV 號或者 App 分享文本時，就會自動觸發解析。

- 支持平台：抖音、B站、快手、小紅書
- 抖音支持 `douyin.com` / `iesdouyin.com`
- B站支持站內連結、`b23.tv`、`BV...`、`av...`
- 快手支持連結同分享口令文本
- 小紅書支持 `xiaohongshu.com` / `xhslink.com`

<Callout type="info">
如果你喺 WebUI 入面關閉咗「默認解析（`videoTool`）」，插件就會按你設定嘅「自定義優先級（`priority`）」工作。
</Callout>

### 手動解析

如果自動解析關閉咗，或者你想處理歷史消息，引用目標消息後發送：

```bash
#解析
#kkk解析
```

<Callout type="info">
`#弹幕解析` 都係引用解析命令，但只對抖音 / B站生效，並且會強制進入彈幕燒錄流程。
</Callout>

## 常用命令

| 命令 | 作用 |
|---|---|
| `#kkk帮助` | 查看幫助菜單 |
| `#kkk版本` / `#kkk更新日志` | 查看當前版本更新日誌 |
| `#kkk更新` | 執行插件更新 |
| `#kkk解析统计` | 查看當前群解析統計 |
| `#kkk全局解析统计` | 查看全局解析統計，只限主人 |
| `#抖音登录` | 獲取抖音 Cookie |
| `#B站登录` | 獲取 B站 Cookie |

## 動態推送

### 訂閱與取消

```bash
#设置抖音推送 抖音号
#设置B站推送 UID
```

- 抖音使用抖音號，唔係暱稱
- B站使用數字 UID
- 兩條命令都係切換式：已訂閱就取消，未訂閱就新增
- 實際權限由「邊個可以設置推送（`push.permission`）」控制

### 推送列表

```bash
#抖音推送列表
#B站推送列表
```

### 強制推送與維護

```bash
#抖音强制推送
#B站强制推送
#抖音全部强制推送
#B站全部强制推送
#kkk设置推送机器人 BotID
#测试抖音推送 https://...
```

- `强制推送` 只處理當前群
- `全部强制推送` 會模擬一次全局定時任務
- `#kkk设置推送机器人` 用來批量改綁定嘅機器人帳號
- `#测试抖音推送` 適合調試推送圖樣式同連結二維碼

## 掃碼登錄與權限

為咗攞到更高畫質、推送能力同更穩定嘅接口權限，建議盡早配置 Cookie。

```bash
#抖音登录
#B站登录
```

<Callout type="info">
呢兩條命令邊個可以用，取決於各平台配置頁入面嘅「邊個可以觸發掃碼登錄（`loginPerm`）」，修改之後通常要重啟。
</Callout>

<Callout type="warn">
掃碼登錄攞到嘅 Cookie 會自動寫入配置。佢哋只會用嚟請求平台官方接口，唔會被插件上傳去第三方。
</Callout>

## 統計與排障

```bash
#kkk解析统计
#kkk全局解析统计
```

統計功能會按群、按平台記錄解析次數，亦會統計活躍用戶同全局趨勢。
