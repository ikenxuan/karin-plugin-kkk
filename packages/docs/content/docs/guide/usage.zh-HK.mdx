## 解析功能

karin-plugin-kkk 嘅核心功能係解析各平台嘅媒體連結。

### 自動解析

預設情況下，插件會監聽群傾偈消息。當消息中包含支持嘅平台連結時，插件會自動觸發解析流程。

- **支持平台**：抖音、B站、快手、小紅書
- **觸發方式**：直接發送包含連結嘅消息（支持混合文本）

<Callout type="info">
部分短連結可能需要重定向才能獲取真實 ID，插件會自動處理呢啲重定向。
</Callout>

### 手動解析

如果自動解析功能被關閉，或者你需要解析一條歷史消息/引用消息，可以使用手動解析命令。

- **命令**：`#解析` 或 `#kkk解析`
- **用法**：引用（回覆）包含連結嘅消息，並發送上述命令。

## 動態推送

訂閱博主/UP主嘅動態更新，並喺第一時間推送到群傾偈。

### 訂閱管理

通过簡單嘅指令即可訂閱或取消訂閱。

<Tabs items={['抖音', 'B站']}>
  <Tab value="抖音">
    **訂閱/取消訂閱**
    ```bash
    #设置抖音推送 + 抖音號
    ```
    *示例：`#设置抖音推送 yuanshen_mihoyo`*

    **說明**：
    - 需要使用博主嘅 **抖音號**（非暱稱）。
    - 命令係**切換式**嘅：如果該群未訂閱，則訂閱；如果已訂閱，則取消訂閱。
  </Tab>
  <Tab value="B站">
    **訂閱/取消訂閱**
    ```bash
    #设置B站推送 + UID
    ```
    *示例：`#设置B站推送 401742377`*

    **說明**：
    - 需要使用UP主嘅 **UID**（數字ID）。
    - 命令係**切換式**嘅：如果該群未訂閱，則訂閱；如果已訂閱，則取消訂閱。
  </Tab>
</Tabs>

### 推送列表

查看當前群傾偈已訂閱嘅所有推送源。

```bash
#抖音推送列表
#B站推送列表
```

### 強制推送

手動觸發一次推送任務檢查，用於測試或立即獲取更新。

```bash
#抖音强制推送
#B站强制推送
#抖音全部强制推送
#B站全部强制推送
```

- **普通強制推送**：僅檢查並推送當前群嘅訂閱。
- **全部強制推送**：模擬一次全局定時任務，檢查所有群嘅訂閱。

## 系統設置

### 掃碼登錄

為了獲取更高畫質嘅影片（如B站1080P+）或使用推送功能，建議配置 Cookie。插件提供了便捷嘅掃碼登錄功能。

```bash
#抖音登录
#B站登录
```

<Callout type="info">
如果唔方便使用掃碼登錄，或需要配置其他平台嘅 Cookie，請參考 [獲取 Cookies 指南](/zh-HK/docs/advanced/ck)。
</Callout>

<Callout type="warn">
掃碼登錄獲取嘅 Cookie 會自動保存。Cookies 僅用於請求平台官方 API，唔會上傳到任何第三方。
</Callout>

### 開關控制

通过指令快速開啟或關閉特定平台嘅推送功能（全局生效）。

```bash
#设置抖音推送开启
#设置抖音推送关闭
#设置B站推送开启
#设置B站推送关闭
```

<Callout type="info">
修改開關狀態後，可能需要重啟插件或等待下一次心跳檢測生效。
</Callout>
