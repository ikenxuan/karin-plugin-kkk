## 解析功能

karin-plugin-kkk 的核心功能是解析各平台的媒体链接。

### 自动解析

默认情况下，插件会监听群聊消息。当消息中包含支持的平台链接时，插件会自动触发解析流程。

- **支持平台**：抖音、B站、快手、小红书
- **触发方式**：直接发送包含链接的消息（支持混合文本）

<Callout type="info">
部分短链接可能需要重定向才能获取真实 ID，插件会自动处理这些重定向。
</Callout>

### 手动解析

如果自动解析功能被关闭，或者你需要解析一条历史消息/引用消息，可以使用手动解析命令。

- **命令**：`#解析` 或 `#kkk解析`
- **用法**：引用（回复）包含链接的消息，并发送上述命令。

## 动态推送

订阅博主/UP主的动态更新，并在第一时间推送到群聊。

### 订阅管理

通过简单的指令即可订阅或取消订阅。

<Tabs items={['抖音', 'B站']}>
  <Tab value="抖音">
    **订阅/取消订阅**
    ```bash
    #设置抖音推送 + 抖音号
    ```
    *示例：`#设置抖音推送 yuanshen_mihoyo`*

    **说明**：
    - 需要使用博主的 **抖音号**（非昵称）。
    - 命令是**切换式**的：如果该群未订阅，则订阅；如果已订阅，则取消订阅。
  </Tab>
  <Tab value="B站">
    **订阅/取消订阅**
    ```bash
    #设置B站推送 + UID
    ```
    *示例：`#设置B站推送 401742377`*

    **说明**：
    - 需要使用UP主的 **UID**（数字ID）。
    - 命令是**切换式**的：如果该群未订阅，则订阅；如果已订阅，则取消订阅。
  </Tab>
</Tabs>

### 推送列表

查看当前群聊已订阅的所有推送源。

```bash
#抖音推送列表
#B站推送列表
```

### 强制推送

手动触发一次推送任务检查，用于测试或立即获取更新。

```bash
#抖音强制推送
#B站强制推送
#抖音全部强制推送
#B站全部强制推送
```

- **普通强制推送**：仅检查并推送当前群的订阅。
- **全部强制推送**：模拟一次全局定时任务，检查所有群的订阅。

## 系统设置

### 扫码登录

为了获取更高画质的视频（如B站1080P+）或使用推送功能，建议配置 Cookie。插件提供了便捷的扫码登录功能。

```bash
#抖音登录
#B站登录
```

<Callout type="info">
如果不方便使用扫码登录，或需要配置其他平台的 Cookie，请参考 [获取 Cookies 指南](/docs/guide/ck)。
</Callout>

<Callout type="warn">
扫码登录获取的 Cookie 会自动保存。Cookies 仅用于请求平台官方 API，不会上传到任何第三方。。
</Callout>

### 开关控制

通过指令快速开启或关闭特定平台的推送功能（全局生效）。

```bash
#设置抖音推送开启
#设置抖音推送关闭
#设置B站推送开启
#设置B站推送关闭
```

<Callout type="info">
修改开关状态后，可能需要重启插件或等待下一次心跳检测生效。
</Callout>
