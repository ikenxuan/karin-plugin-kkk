本插件採用 **React + TailwindCSS** 技術棧進行圖片渲染，將現代前端嘅工程化能力引入模板開發。

相比傳統模板引擎在維護上嘅痛點——邏輯與視圖耦合、全局 CSS 衝突、缺乏類型檢查——本方案通過**組件化構建**、**原子化樣式**同**類型安全**三條路徑徹底解決咗呢啲問題。

<Callout type="warn" title="傳統方案嘅維護陷阱">
基於 `art-template` 嘅模板喺項目迭代中往往會陷入維護泥潭：
- **邏輯黑洞**：業務邏輯混雜喺 HTML 模板中，難以閱讀同剝離。
- **樣式衝突**：全局 CSS 類名隨時間推移不斷堆積，修改一處可能導致多處崩壞。
- **重構風險**：缺乏類型檢查，修改字段名就好似「排雷」，只能祈禱運行時唔出錯。
</Callout>

整個渲染鏈路分為四層：

1. **數據層（`@kkk/richtext`）**：定義平台無關嘅富文本文檔協議，作為 core 同 template 之間嘅數據邊界。
2. **Core 解析層**：將抖音、B 站等平台嘅原始 API 響應解析為標準嘅 `RichTextDocument` JSON。
3. **Template 渲染層**：React 組件消費 JSON 數據，TailwindCSS 處理樣式，最終通過 SSR 輸出靜態 HTML。
4. **輸出層**：Karin 框架調用 Puppeteer 截圖，生成圖片消息發送到群聊。

## 富文本文檔（RichTextDocument）

`@kkk/richtext` 係 core 包同 template 包之間嘅共享子包，負責定義**平台無關嘅富文本中間表示（IR）**。

<Callout type="info" title="設計動機">
唔同平台嘅文本描述格式差異好大：抖音嘅富文本係一段帶特殊標記嘅字符串，B 站就提供咗結構化嘅內容節點。如果喺 template 側分別解析，會導致平台邏輯滲漏到渲染層，維護成本極高。

富文本文檔將呢個問題收攏到 core 層：core 負責將各平台嘅異構數據轉換成統一嘅 JSON 節點樹，template 只負責渲染。
</Callout>

### 節點類型

富文本文檔支持兩類節點：**行內節點**同**塊級節點**。

**行內節點**用於描述段落內嘅文本片段：

| 節點類型 | 用途 | 示例 |
| :--- | :--- | :--- |
| `text` | 普通文本，支持粗體/斜體/顏色/超連結 | 評論正文 |
| `emoji` | 平台表情包圖片 | [doge] |
| `mention` / `at` | @用戶 | @某某 |
| `searchKeyword` | 搜索詞高亮 | 帶搜索圖標嘅藍色高亮文本 |
| `topic` | 話題標籤 | #某某話題# |
| `lottery` | 抽獎資訊 | 帶抽獎圖標嘅文本 |
| `webLink` | 網頁連結 | 帶連結圖標嘅標題 |
| `vote` | 投票 | 帶投票圖標嘅標題 |
| `viewPicture` | 查看圖片提示 | 帶相冊圖標嘅提示文本 |
| `lineBreak` | 換行 | `<br />` |

**塊級節點**用於描述文檔結構：

| 節點類型 | 用途 |
| :--- | :--- |
| `heading` | 標題（1-6 級） |
| `paragraph` | 段落 |
| `image` | 圖片 |
| `blockquote` | 引用塊 |
| `list` / `listItem` | 有序/無序列表 |
| `codeBlock` | 代碼塊（帶語法高亮同行號） |
| `linkCard` | 連結卡片 |

### 喺 core 中使用

`@kkk/richtext` 嘅 `parse` 模組提供咗一系列工廠函數，用於從平台原始數據中構建節點：

```ts twoslash
import { createTextNode, createEmojiNode, createParagraphNode, createRichTextDocument } from '@kkk/richtext'

const doc = createRichTextDocument([
  createParagraphNode([
    createTextNode('呢條片太正喇', { bold: true }),
    createEmojiNode('doge', 'https://example.com/doge.png'),
    createTextNode('！推薦大家睇下。')
  ])
], { platform: 'douyin' })
```

core 側只需要導入**類型**同**節點創建方法**，唔依賴 React 運行時，輸出嘅係可序列化嘅純 JSON。

### 喺 template 中使用

`@kkk/richtext` 嘅 `react` 模組提供咗渲染器，將 `RichTextDocument` 轉成 React 節點：

```tsx
import { renderRichTextToReact } from '@kkk/richtext/react'

// 喺平台模板組件中
const content = renderRichTextToReact(document, {
  mention: { className: 'text-blue-500' },
  topic: { className: 'text-pink-500' },
  searchKeyword: { className: 'text-blue-600 bg-blue-100', iconClassName: 'text-blue-400' },
})

return <div className="text-base leading-relaxed">{content}</div>
```

渲染器會自動處理以下細節：
- **文本轉義**：React 自動轉義文本內容，防止 XSS。
- **圖片安全**：對 `emoji` 同 `image` 節點嘅 `src` 做協議白名單校驗（僅允許 `http://`、`https://` 同 `data:image/*;base64`）。
- **URL 自動識別**：`text` 節點中嘅連結會被自動包裹為可點擊嘅超連結。
- **行內樣式**：支持 `bold`、`italic`、`strike`、`color`、`fontSize`、`link` 等行內樣式。

<Callout type="info" title="點解唔用 dangerouslySetInnerHTML？">
平台返回嘅富文本描述往往包含未經校驗嘅 HTML。使用 `renderRichTextToReact` 將結構化 JSON 映射為 React 節點，可以避免直接注入 HTML 帶嚟嘅安全風險，同時讓樣式完全由 Tailwind 接管。
</Callout>

### 節點歸一化

`createRichTextDocument` 內部會自動調用 `normalizeRichTextNodes`，合併相鄰嘅文本節點並丟棄空文本節點。咁樣 core 喺解析時可以按匹配過程簡單 `push` 節點，唔使擔心碎片過多嘅問題。

## 開發工作流

內置嘅**可視化開發面板** (`pnpm dev`) 大幅提升模板開發效率：

- **實時預覽**：修改組件代碼，瀏覽器即時刷新渲染結果（HMR 熱更新）。
- **數據 Mock**：可配置多套 JSON 數據，輕鬆測試文本超長、頭像缺失等邊緣情況。
- **快速調試**：無需啟動成個 Bot，僅需瀏覽器即可完成模板開發。

**入口與初始化**：`reactServerRender` 接收渲染請求、輸出目錄同插件數組，首先確保輸出目錄存在，然後通過 `ComponentAutoRegistry` 掃描配置文件，按平台分組懶加載組件模塊並綁定數據驗證函數，最終存入 Map 註冊表。

**SSRRender 實例化**：創建 `ResourcePathManager` 檢測運行環境（開發/生產），解析包目錄路徑並處理 pnpm 符號鏈接；創建 `HtmlWrapper` 負責 HTML 包裝；創建 `PluginContainer` 並按 `enforce` 字段（pre → normal → post）對插件排序。

**渲染流程**：構建 `PluginContext` 上下文同 `RenderState` 狀態對象 → 執行 `runBefore` 前置插件（如二維碼生成）注入額外 props → `ComponentRendererFactory` 從註冊表查找組件、驗證數據、合併 props、處理嵌套路徑、調用 `React.createElement` → 執行 `runDuring` 渲染插件（可包裝組件）→ 調用 `renderToString` 將 React 組件轉為 HTML 字符串 → 執行 `runAfter` 後置插件（可修改 HTML）。

**輸出階段**：生成帶時間戳嘅安全文件名 → `HtmlWrapper.wrapContent` 計算 CSS 同圖片嘅相對路徑、替換圖片 src、注入 DOCTYPE/meta/CSS link/主題類名 → 寫入文件系統 → 返回 HTML 文件路徑供 Karin 框架截圖。

<Callout type="info" title="性能優化">
SSR 直接輸出包含完整樣式同內容嘅 HTML，Puppeteer 打開頁面後**無需等待 JavaScript 加載同執行**即可立即截圖，顯著降低咗渲染耗時同內存佔用。
</Callout>

## 生態復用

得益於 React 標準，可以直接引入成熟嘅庫嚟豐富**靜態畫面**嘅表現力：

- **排版佈局**：引入現代化嘅 UI 組件庫，快速構建精美嘅卡片、列表。
- **數據可視化**：使用專業圖表庫將複雜數據渲染為統計圖表。
- **矢量圖標**：引入海量 SVG 圖標庫，支持任意縮放不失真。
