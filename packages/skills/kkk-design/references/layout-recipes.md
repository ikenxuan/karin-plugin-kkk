# Layout Recipes

## 固定画布基础

```tsx
<main className="relative overflow-hidden" style={{ width: 1440, minHeight: 1800 }}>
  <Background />
  <section className="relative z-10 px-20 py-18">...</section>
</main>
```

- 根节点稳定宽度，截图缩放交给外层渲染器。
- 所有绝对定位背景放在 z-0 或负层级，正文放 z-10。
- 背景装饰使用 `pointer-events-none` 和 `select-none`。

## 克制内容详情

适合作者 + 内容 + 媒体 + 统计。

1. 外层 `px-20 py-16`。
2. 头部 `flex items-center gap-6`，头像 128-160px。
3. 作者名 56-72px，时间/来源 32-40px。
4. 主标题 64-80px；正文 48-60px，行高 1.6。
5. 媒体块全宽，圆角 32-40px，顶部或底部可叠加信息条。
6. 底部统计 `grid` 或 `flex`，每项 32-48px 字号。

## 克制回复树

适合多层文本关系。

1. 根项使用两列：头像列 120-160px，内容列自适应。
2. 子项头像缩小到 80-110px，字号降 10%-20%。
3. 用 1-2px 竖线表达层级，颜色 `border/40`。
4. 曲线连接只在必要时使用，避免线条抢正文。
5. 深度超过 5-6 层时折叠或提示剩余数量。

## 图片组

- 单图：全宽或 90%-95% 宽，圆角 32-40px。
- 2 图：双列等宽，gap 12-20px。
- 3/6/9 图：三列方图，外侧角更大，内部角较小。
- 瀑布流：两列，gap 16px，适合纵向图片。
- 堆叠预览：前景图清晰，后两层偏移 16/32px，降低亮度和透明度。

## 双栏弥散海报

适合一侧身份/说明、一侧主内容。

```txt
12 columns
left: 4 columns
right: 8 columns
gap: 48-64px
padding: 48-96px
```

- 左侧放系统标签、头像/图标、摘要、二维码、辅助说明。
- 右侧放主媒体、主面板、统计或行动区域。
- 右侧面板可使用 `bg-surface/60 backdrop-blur-xl rounded-[3rem] p-10`。
- 左侧不要塞满，保留大留白和装饰平衡。

## 纵向报告

适合数据总览、趋势、分布、排行。

1. 顶部：系统标签 + 巨型标题 + 一句话结论。
2. 总览：2-3 张关键数字卡，数字 72-112px。
3. 趋势：单个大图表面板，不与排行挤在同一行。
4. 分布：环图/饼图 + 图例双栏。
5. 排行：每行头像/名称/条形/数值，第一名可加奖章或强调色。
6. 底部：来源、时间、版本或简短说明。

## 中央行动页

适合二维码、确认码、一次性操作、强提醒。

1. 画布 1440px 宽，高度 1800-2200px。
2. 背景光集中在中轴，上中下各一层。
3. 顶部标签 24-32px，主标题 90-140px。
4. 核心物件居中，QR 720-840px 或代码/令牌超大显示。
5. 警示条紧贴核心物件上方或下方，红色只用于风险。
6. 说明分为 2 个左右短模块，每个模块不超过 3 行正文。

## 大媒体背景

适合媒体封面、直播/状态、强氛围详情。

1. 背景图全屏铺满，`scale(1.5)`，blur 120px，saturate 1.5-1.8。
2. 叠加渐变遮罩，保证文字区域对比。
3. 噪点 overlay 0.35-0.45。
4. 前景内容从上到下：状态标签、标题、描述、作者/来源、统计、QR。
5. 关键文字使用 64-90px，避免压在高复杂度背景上。

## 列表看板

- 背景使用弥散海报，列表卡片使用克制内容卡片。
- 卡片 `bg-surface/60 backdrop-blur-xl border border-border/50 rounded-4xl`。
- 头像/封面保持一致尺寸；标题两行内；统计固定位置。
- 行数太多时优先分页或截断，不要把字号压小。

## 时间线

- 左侧日期/节点，右侧内容卡。
- 日期数字 64-96px，月份/星期 24-36px。
- 竖线 2-4px，透明度 0.30-0.50。
- 内容卡圆角 24-32px，图片和标题并列时保持固定比例。
